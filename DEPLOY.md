# Deploy SK Mobile Shop

One-click style deploy: **API on Render** + **Web on Netlify**.

Default login (seeded on API start):

| Field | Value |
|-------|--------|
| Email | `skmobile@gmail.com` |
| Password | `SK@7869123` |

---

## 1. Prerequisites

1. GitHub repo with this project pushed.
2. **Supabase** (or any Postgres) — copy the **Session pooler** `DATABASE_URL`.
3. Encode special characters in the password (`@` → `%40`, `#` → `%23`).

---

## 2. Backend — Render (Blueprint)

1. [Render Dashboard](https://dashboard.render.com/) → **New** → **Blueprint**.
2. Connect your GitHub repo (uses root `render.yaml`).
3. Before deploy finishes, set **Environment** for `sk-mobile-api`:

   | Key | Value |
   |-----|--------|
   | `DATABASE_URL` | Your Postgres connection string |
   | `CORS_ORIGIN` | Your Netlify URL (set after step 3), e.g. `https://skmobile.netlify.app` |

   `JWT_SECRET` is auto-generated by the blueprint.

4. Deploy. On each start the API runs:
   - `prisma migrate deploy`
   - `prisma db seed` (upserts owner user)
   - Express server on port `10000`

5. Copy the live API URL, e.g. `https://sk-mobile-api.onrender.com`.
6. Test: `https://YOUR-API.onrender.com/health` → `{ "status": "ok", "db": "connected" }`.

---

## 3. Frontend — Netlify

1. [Netlify](https://app.netlify.com/) → **Add new site** → **Import from Git**.
2. Select the same repo (Netlify reads root `netlify.toml`).
3. **Site configuration** → **Environment variables**:

   | Key | Value |
   |-----|--------|
   | `NEXT_PUBLIC_API_URL` | `https://sk-mobile-api.onrender.com` (your Render URL, no trailing slash) |

4. Deploy site.
5. Copy Netlify URL (e.g. `https://random-name.netlify.app`).

---

## 4. Link Render ↔ Netlify

1. Render → `sk-mobile-api` → **Environment** → set `CORS_ORIGIN` to your **exact Netlify URL**.
2. **Save** → Render redeploys automatically.
3. Open Netlify URL → login with seeded credentials above.

---

## 5. Local vs production

| | Local | Production |
|---|--------|------------|
| API | `http://localhost:4000` | Render URL |
| Web | `http://localhost:3000` | Netlify URL |
| Env | `.env` / `.env.local` | Platform dashboards only |

Do **not** commit `.env` files. Use `.env.example` as reference.

---

## 6. Troubleshooting

| Issue | Fix |
|-------|-----|
| Login fails | Check `NEXT_PUBLIC_API_URL` on Netlify matches Render URL |
| CORS error | `CORS_ORIGIN` on Render must match Netlify URL exactly |
| API 503 health | `DATABASE_URL` wrong or Supabase paused |
| Build fails on Render | Ensure `DATABASE_URL` is set before deploy; check Render logs |
| Netlify build fails | Node 20; run `npm ci && npm run build` locally from repo root |

---

## 7. Manual commands (optional)

```bash
# From repo root
npm ci
npm run build --workspace=@sk-mobile/shared
npm run db:generate --workspace=@sk-mobile/api
npm run build --workspace=@sk-mobile/api
npm run db:migrate --workspace=@sk-mobile/api
npm run db:seed --workspace=@sk-mobile/api
```
